# Reverse Playbook

## 分析清单

1. 目标与边界
- 目标 URL、目标页面、目标流程。
- 允许分析范围（仅公开页面/含登录后流程）。
- 法务与合规边界（不抓取隐私数据，不复用版权素材）。

2. 页面与交互
- 页面信息架构（导航、模块、入口、回流路径）。
- 关键交互（筛选、搜索、提交、支付、异步加载）。
- 状态覆盖（初始化、加载中、成功、空态、失败态）。

3. 网络与数据
- 核心请求列表（方法、路径、参数、返回结构）。
- 鉴权与会话机制（cookie、token、刷新策略）。
- 分页、排序、过滤、缓存与重试策略。

4. 技术栈推断
- 前端框架与构建工具线索。
- UI 组件库与状态管理模式线索。
- 第三方服务（统计、监控、图床、CDN、支付）。

5. 复刻设计
- 功能模块拆分（MVP/增强项）。
- 数据模型与接口契约草案。
- 前端组件树与页面路由草案。
- 非功能需求（性能、安全、可观测性）。

## 常见陷阱

1. 只看静态页面不看请求，导致方案无法落地。
2. 仅复制 UI，不抽象数据模型与状态流。
3. 没有先做 MVP 边界，直接进入全量复刻。
4. 忽略鉴权和风控导致联调失败。
